About i18n

Internationalization (i18n) is the process of designing and preparing your app to be usable in different languages. In this section, you can find examples of how to use i18n in your app.

Metro UI i18n module

Metro UI has a built-in i18n module that helps you to easily translate your app into different languages and switch between defined languages. Locales out from box - en, ua. Available built-in labels you can see in Metro UI source code on GitHub.

Methods

You can use the following methods to work with the i18n module (Metro.i18n scope):

  • load(lang = 'en') - load language file to current locale
  • add(lang, data) - add locale
  • get(key, lang) - get specified label from locale
  • updateUI(context, lang) - update labels for specified context and language, by default, this method uses document as context and attr lang from html tag
  • extend(data) - extend locales with custom labels
Properties
  • Metro.i18n.language - current language ID
  • Metro.locales - available locales
  • Metro.locale - current locale - is an object contains pairs key, value
How it works

When Metro UI is initialized, it loads the language, defined in attr lang from html tag or en if attribute is not specified. Each Metro UI component hash a property strings, which contains labels for the current loaded language. You can use attr lang for any HTML element to specify the language for this element and its children.

Extends

You can extend the Metro UI i18n module with custom labels for your app. To do this, you need to call the Metro.i18n.extend(data) method and pass an object with custom labels.

Update UI

To set i18n labels, add attr data-i18n to the HTML element and specify the label key. To update labels for the specified context and language, you need to call the Metro.i18n.updateUI(context, lang) method. By default, this method uses document as context and attr lang from html tag.
